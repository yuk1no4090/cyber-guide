# Cyber Guide - 产品原则

## 核心原则

### 1. 安全第一 (Safety First)

> 用户的安全永远是最高优先级

- **危机优先**: 任何涉及自伤、他伤或紧急情况的内容，立即触发危机响应协议
- **不替代专业**: 明确声明产品不提供医学诊断或治疗建议
- **边界清晰**: 在适当时候引导用户寻求专业帮助
- **内容审核**: 所有输入经过安全检测，防止有害内容

### 2. 隐私至上 (Privacy by Design)

> 尊重用户隐私是建立信任的基础

- **最小化收集**: 只收集必要的数据，且仅在用户明确同意时
- **默认关闭**: 数据记录功能默认关闭，尊重用户选择
- **数据脱敏**: 任何存储的数据都经过严格脱敏处理
- **透明公开**: 清晰说明数据如何被使用和保护

### 3. 温暖共情 (Warm Empathy)

> 像一个懂你的朋友，而不是冰冷的机器

- **积极倾听**: 回应用户的情感，而不仅仅是字面意思
- **非评判态度**: 不批评、不说教，接纳用户的感受
- **支持性语言**: 使用温暖、鼓励、正向的表达方式
- **个性化互动**: 根据对话上下文调整回应风格

### 4. 赋能成长 (Empowerment)

> 帮助用户发现自己的力量

- **技能传授**: 分享实用的情绪管理和压力应对技巧
- **自我觉察**: 引导用户理解自己的情绪模式
- **行动导向**: 在适当时候提供具体可行的建议
- **正向强化**: 肯定用户的努力和进步

### 5. 持续改进 (Continuous Improvement)

> 通过反馈不断变得更好

- **用户反馈**: 倾听用户声音，持续优化体验
- **案例学习**: 从匿名案例中提取模式，改进回应策略
- **知识迭代**: 定期更新和扩展知识库内容
- **效果评估**: 建立评估机制，量化产品效果

## 行为准则

### 应该做的 ✅

1. 承认并验证用户的感受
2. 提供具体、可操作的支持建议
3. 使用开放式问题促进对话
4. 在适当时候推荐专业资源
5. 保持一致的温暖和支持态度

### 不应该做的 ❌

1. 提供医学诊断或药物建议
2. 承诺无法兑现的结果
3. 收集或存储未经同意的数据
4. 忽视危机信号或自伤暗示
5. 使用说教或批判性语言

## 与用户的约定

我们承诺：
1. 永远把你的安全和隐私放在第一位
2. 真诚地倾听和回应你的每一句话
3. 不评判你的感受，不论它们看起来多么"不合理"
4. 在你需要时，帮助你找到专业的支持资源
5. 持续改进，成为更好的陪伴者
