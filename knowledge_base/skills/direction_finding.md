# 找方向：别想了，去试

## 最大的误区

很多人觉得"找方向"是一个思考题——坐在那里分析利弊、看看知乎、问问学长，想明白了再行动。

但真相是：**方向不是想出来的，是试出来的。**

你坐在桌前纠结"前端还是后端还是算法还是产品经理"，跟你没试过任何一个就纠结"北京还是上海还是深圳"一样——没有体感的分析都是空转。

## 我自己的经历

我本科学的软件工程，大三的时候不知道该选什么方向。前端？后端？算法？AI？产品经理？

我做了一件事：每个方向都花了 2-3 周时间去"沾一下"：

- 写了一个简单的前端页面——感觉还行但不够兴奋
- 用 Python 写了几个小脚本——能做但没什么成就感
- 试着用 AI API 做了个小工具——突然觉得"嗯，这个有意思"
- 跟一个做产品的朋友聊了几次——发现自己对"为什么要做这个功能"的问题更感兴趣

最后我倾向了 AI 产品经理的方向。不是因为我"分析出来"这条路最好，而是因为我在实际接触中**感受到了兴趣和能量**。

到现在我也不能百分百确定这个方向是对的。但至少我是在"有体感"的基础上做的选择，不是凭空猜。

## 一套最小试错系统

### 第一步：列出你的候选方向（不超过5个）

不需要精确，粗略就行。比如：
- 前端开发
- 后端/系统开发
- 算法/AI
- 产品经理
- 数据分析

### 第二步：每个方向做一个"两周实验"

给每个方向 2 周时间。在这 2 周里：

1. **做一个最小项目**：不是做一个完整产品，是做一个能跑的 demo。前端做一个小页面，后端写一个 API，算法刷 20 道题，产品写一份需求文档。
2. **找一个这个方向的人聊 30 分钟**：问 ta 日常工作是什么样的、最爽和最烦的部分是什么。
3. **记录你的感受**：不是分析利弊，而是记录你做这件事时的**身体感觉**——是觉得时间过得很快还是很慢？是做完有成就感还是如释重负？

### 第三步：比较"能量值"而不是"前景分析"

2 周结束后，比较每个方向给你带来的"能量"。哪个方向让你：
- 做的时候不觉得累？
- 做完之后想继续做？
- 跟别人聊的时候眼睛发亮？

那个方向大概率就是你该走的。

### 第四步：选一个，投入 3 个月

选了就不要回头看。投入 3 个月的深度时间——做一个完整项目、拿一个证书、找一段实习。3 个月之后你对这个方向的理解会完全不同。

如果 3 个月后发现不对，再调也来得及。大部分方向之间的技能是可以迁移的。

## 几个要避开的坑

- **"等我准备好了再开始"**：你永远不会准备好，因为你不知道需要准备什么。直接开始，边做边学。
- **"我什么都想学"**：什么都学 = 什么都不精 = 什么都不能拿来说事。先选一个方向挖到足够深，再横向扩展。
- **"这个方向没前途"**：在 CS 领域，几乎所有方向都有饭吃。差别在于你能不能做到前 20%。选一个你愿意花时间的方向，比选一个"前途好"但你不想做的方向重要得多。
- **"别人都在做 XX"**：别人做什么跟你没关系。你是你，你的兴趣、能力、性格跟他们不一样。

## 说到底

找方向不是一次性决定，是一个持续校准的过程。你不需要今天就想清楚未来十年，你只需要想清楚**下一个 3 个月做什么**。

然后去做。做着做着，方向就出来了。
